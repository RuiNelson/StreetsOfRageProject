# Tools User Manual

This is a simple user manual for the unified `tools` command.

Run every command from the repository root:

```bash
python3 -m tools <command> [options]
```

To list the available commands:

```bash
python3 -m tools --help
```

## Available Commands

| Command | Purpose |
| --- | --- |
| `disassemble` | Disassemble the 68000 ROM to assembly and optionally write a coverage map. |
| `recompile` | Recompile the ROM to C++ (`Sor.hpp` / `Sor.cpp`). |
| `speculative-scan` | Find candidate entry points in ROM regions that are still unknown. |
| `remove-data` | Remove assembly blocks that contain data directives. |
| `label-diff` | Print labels that exist in the second `.asm` file but not in the first. |
| `map-label-gaps` | Print reference labels that still point to `X` bytes in the ROM map. |
| `iterative-disasm` | Run the iterative loop that adds missing addresses to an aux file. |

Accepted aliases:

| Alias | Real command |
| --- | --- |
| `disasm` | `disassemble` |
| `recompiler` | `recompile` |
| `specscan` | `speculative-scan` |
| `strip-data` | `remove-data` |
| `labels-missing` | `label-diff` |

## disassemble

Disassembles the ROM into 68000 assembly.

Usage:

```bash
python3 -m tools disassemble <rom> -o <output.asm> [options]
```

Typical project example:

```bash
python3 -m tools disassemble rom/SOR.bin \
    -o output/sor.asm \
    -a code-analysis/aux_addresses.txt \
    --addresses-csv code-analysis/addresses.csv \
    --blocks-csv code-analysis/blocks.csv \
    --labels-csv code-analysis/labels.csv \
    --map output/sor.map \
    -v
```

Arguments and options:

| Option | Required | Description |
| --- | --- | --- |
| `<rom>` | Yes | Path to the ROM binary. |
| `-o, --output <file>` | Yes | Assembly output file. |
| `-a, --aux <file>` | No | Auxiliary entry-point file, one hexadecimal address per line. |
| `--addresses-csv <file>` | No | CSV in `address,label,comment` format for address labels and comments. |
| `--blocks-csv <file>` | No | CSV in `start,end,name` format for memory blocks used in offset resolution. |
| `--labels-csv <file>` | No | CSV containing code segment labels. |
| `--map <file>` | No | Write a ROM coverage map. |
| `--indirect-jumps <file>` | No | Write unresolved indirect jump/call addresses. |
| `-v, --verbose` | No | Print progress and statistics to stderr. |
| `-h, --help` | No | Show command help. |

The map generated by `--map` uses one byte per ROM byte:

| Byte | Meaning |
| --- | --- |
| `X` | Unknown or data. |
| `S` | Subroutine entry. |
| `s` | Subroutine/handler end (`rts`, `rte`, `rtr`, `illegal`). |
| `L` | Branch/jump target label. |
| `C` | Regular code. |
| `c` | Continuation byte inside a multi-byte instruction. |

## recompile

Generates C++ from the ROM by running the disassembler internally.

Usage:

```bash
python3 -m tools recompile <rom> [options]
```

Typical example:

```bash
python3 -m tools recompile rom/SOR.bin -o src/generated
```

Example with speculative addresses:

```bash
python3 -m tools recompile rom/SOR.bin \
    -o src/generated \
    --speculative code-analysis/speculative_addresses.txt
```

Arguments and options:

| Option | Required | Default | Description |
| --- | --- | --- | --- |
| `<rom>` | Yes | | Path to the ROM binary. |
| `-o, --out-dir <dir>` | No | `src/generated` | Directory where `Sor.hpp` and `Sor.cpp` are written. |
| `--aux <file>` | No | `code-analysis/aux_addresses.txt` | Known auxiliary entry points. |
| `--speculative <file>` | No | empty | Speculative entry points to compile with confirmation hooks. |
| `--labels-csv <file>` | No | `code-analysis/labels.csv` | CSV of code segment names. |
| `--addresses-csv <file>` | No | `code-analysis/addresses.csv` | CSV of address labels. |
| `-v, --verbose` | No | | Print more information during recompilation. |
| `-h, --help` | No | | Show command help. |

Notes:

- The generated `Sor.cpp` is self-contained; it emits small one-line cast macros and all 68000 instruction semantics directly.
- If `--speculative` is not provided, no `confirmSpeculative` hooks are emitted.
- If the recompiler finds an unsupported instruction outside the speculative scope, it fails instead of generating a silent stub.

## speculative-scan

Scans ROM regions marked as `X` for possible function entry points.

Usage:

```bash
python3 -m tools speculative-scan <map_file> <rom_file> <aux_file> [options]
```

Example:

```bash
python3 -m tools speculative-scan \
    output/sor.map \
    rom/SOR.bin \
    code-analysis/aux_addresses.txt \
    -o code-analysis/speculative_addresses.txt
```

Arguments and options:

| Option | Required | Default | Description |
| --- | --- | --- | --- |
| `<map_file>` | Yes | | ROM coverage map, usually `output/sor.map`. |
| `<rom_file>` | Yes | | ROM binary. |
| `<aux_file>` | Yes | | Known auxiliary addresses, used to avoid duplicate candidates. |
| `--dry-run` | No | | Print only the candidate count; do not write the output file. |
| `--no-static-filter` | No | | Scan the supplied map directly instead of first applying the recompiler static table-discovery fixpoint. |
| `-o, --output <file>` | No | `code-analysis/speculative_addresses.txt` | Output file. |
| `-h, --help` | No | | Show command help. |

## remove-data

Removes assembly blocks that contain `dc.b`, `dc.w`, or `dc.l`.

Usage:

```bash
python3 -m tools remove-data [asm-file]
```

Arguments:

| Argument | Required | Default | Description |
| --- | --- | --- | --- |
| `[asm-file]` | No | `etc/sor-exodus.asm` | File to modify in place. |
| `-h, --help` | No | | Show command help. |

Example:

```bash
python3 -m tools remove-data output/sor.asm
```

Warning: this command modifies the selected file in place.

## label-diff

Compares two assembly files and prints labels that exist in the second file but not in the first.

Usage:

```bash
python3 -m tools label-diff <file1.asm> <file2.asm>
```

Arguments:

| Argument | Required | Description |
| --- | --- | --- |
| `<file1.asm>` | Yes | Base assembly file. |
| `<file2.asm>` | Yes | Reference/comparison assembly file. |
| `-h, --help` | No | Show command help. |

Example:

```bash
python3 -m tools label-diff output/sor.asm etc/sor-exodus.asm
```

The output is one hexadecimal address per line.

## map-label-gaps

Prints reference assembly labels that still land on `X` bytes in the ROM map.

Usage:

```bash
python3 -m tools map-label-gaps <rom.map> <reference.asm>
```

Arguments:

| Argument | Required | Description |
| --- | --- | --- |
| `<rom.map>` | Yes | Coverage map generated by `disassemble --map`. |
| `<reference.asm>` | Yes | Reference assembly with `loc_XXXXXXXX` or `sub_XXXXXXXX` labels. |
| `-h, --help` | No | Show command help. |

Example:

```bash
python3 -m tools map-label-gaps output/sor.map etc/sor-exodus.asm
```

The output is one hexadecimal address per uncovered label.

## iterative-disasm

Repeats the disassembly process, finds the first map gap, and appends that address to the aux file.

Usage:

```bash
python3 -m tools iterative-disasm \
    <sor.asm> \
    <sor.map> \
    <sor-exodus.asm> \
    <aux_addresses.txt> \
    <rom.bin>
```

Arguments:

| Argument | Required | Description |
| --- | --- | --- |
| `<sor.asm>` | Yes | Assembly generated by the disassembler. |
| `<sor.map>` | Yes | Coverage map generated by the disassembler. |
| `<sor-exodus.asm>` | Yes | Reference/active-disassembly assembly file. |
| `<aux_addresses.txt>` | Yes | File where new entry points are appended. |
| `<rom.bin>` | Yes | ROM binary. |
| `-h, --help` | No | Show command help. |

Example:

```bash
python3 -m tools iterative-disasm \
    output/sor.asm \
    output/sor.map \
    etc/sor-exodus.asm \
    aux_addresses.txt \
    rom/SOR.bin
```

The command prints a table with instruction, subroutine, label, and per-iteration delta counts.

## Common Workflows

Generate assembly and a coverage map:

```bash
./disassemble.sh
```

The script internally calls:

```bash
python3 -m tools disassemble ...
```

Recompile the game:

```bash
python3 -m tools recompile rom/SOR.bin -o src/generated
```

Generate speculative candidates:

```bash
python3 -m tools speculative-scan output/sor.map rom/SOR.bin code-analysis/aux_addresses.txt
```

Compare coverage against the reference:

```bash
python3 -m tools map-label-gaps output/sor.map etc/sor-exodus.asm
```

## Compatibility

The old entry points are still available, for example:

```bash
python3 -m tools.disassembler ...
python3 -m tools.recompiler ...
python3 -m tools.speculative_scan ...
```

For normal use, prefer the unified command:

```bash
python3 -m tools <command> ...
```
