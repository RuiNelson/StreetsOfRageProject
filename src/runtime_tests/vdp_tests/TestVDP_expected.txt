TEST 01 — Registers                               (no PNG, stdout only)
────────────────────────────────────────────────────────────────────────
Console output only.  Check the printed lines:

  status register = 0x34C0   (or similar value with bits [7:6] = 11)
  bits [7:6] always-1: OK

  - The status word must have bits 7 and 6 set (0xC0 mask always set).
  - Bit 9 (FIFO empty) is set → value ≥ 0x3400.
  - Bit 3 (VBlank) may be set depending on timing.
  - FAIL in the last line means the emulator is broken.

DEPTH EFFECT — special note for Test 16
────────────────────────────────────────────────────────────────────────
The duck-behind-girl depth effect uses LINK CHAIN ORDERING, not the
sprite priority bit:

  Duck in front (sin(angle) >= 0):
    SAT link chain: slot 0 (duck) → slot 1..6 (girl) → end
    Duck is processed last by the renderer → drawn on top.

  Duck behind (sin(angle) < 0):
    SAT link chain: slot 0 (off-screen dummy) → slot 1..6 (girl)
                    → slot 7 (duck) → end
    Girl is processed last → girl pixels cover duck pixels.

To verify the depth effect is working, observe the live window during
the animation: the duck should visibly pass behind the girl during the
lower half of its orbit and in front during the upper half.


────────────────────────────────────────────────────────────────────────
COMMON FAILURE MODES
────────────────────────────────────────────────────────────────────────
  Black bar at bottom of any PNG:
    → Framebuffer height exceeds SCREEN_H (224). Should be 320×224.

  All sprites always on top of each other regardless of SAT order:
    → getSpritePixel() ignores link chain order.

  Planes appear identical after VScroll (Test 10):
    → VSRAM write or VScroll read is broken.

  Window HUD scrolls with Plane A (Test 13):
    → isWindowActive() logic is wrong; window is not being checked
      before falling back to Plane A.

  DMA fill produces wrong tile content (Test 04):
    → VRAM fill byte-swap is incorrect (fill byte written to wrong
      address within each word pair).

  Background colour visible everywhere (Test 07 or 15):
    → Plane A priority compositing not working; transparent pixels
      on Plane A should show Plane B, not the backdrop.

  Text invisible or garbled (Test 05):
    → Font DMA transfer wrong, or nametable tile indices off by one.
